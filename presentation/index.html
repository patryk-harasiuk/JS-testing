Javascript testy

Rodzaje testów

Testy jednostkowe

Celem testu jednostkowego jest walidacja wydzielonej części kodu. Unit testy mają wąski zakres dzięki czemu pozwalają przetestować wszystkie możliwe przypadki danego fragmentu kodu.

Testy integracyjne

Testy integracyjne sprawdzają czy oddzielne części aplikacji współpracują ze sobą zgodnie z założeniami, tak samo jak będzie to miało miejsce w środowisku produkcyjnym.

Testy funkcjonalne / E2E

Testy end-to-end realizują skomplikowane scenariusze uzycia aplikacji, zwykle wymagają dostępu do wszystkich zasobów aplikacji produkcyjnej - takich jak baza danych czy serwer.

Koncepty

  • Matchers
  • Mock
  • Lifecycle

Matchers

Matchery pozwalają sprawdzić czy wyniki testów są zgodne z założeniami 


				test('factorial of 2', () => { 
					expect(factorial(2)).toBe(2);
				});

Mocking

Testowany fragment kodu moze być zalezny od innych elementów. Aby przetestować daną funkcjonalność w izolacji, możemy zastąpić inne elementy od których jest zalezny ich atrapami.

Lifecycle

Podczas testowania często wykonywane jest wiele testów i kazdy z nich wymaga wykonania pewnych akcji przed uruchomieniem testu. Większość framweworków zapewnia funkcje pomocnicze dla ulatwienia tego procesu. 


				beforeEach(() => {
					// Initialize objects
				});
				  
				afterEach(() => {
					// Tear down objects
				});
